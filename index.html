핵심 요구사항 분석

  • 3-actor 반응형 웹 통합 설계 (의뢰인 / 행정사 / 관리자)
  • 카테고리(음주운전·인허가·비자 등) 기반 의뢰서 작성 + 멀티파트 파일 업로드
  • 수신된 행정사 견적 비교함 (금액·프로필·리뷰 별점 요약)
  • 매칭 전 1:1 메시지(채팅/문의) 송수신, 매칭 후 별점 + 텍스트 리뷰
  • 행정사 사업자등록증·자격증 업로드 → 관리자 승인/반려 워크플로우
  • 전문 분야 매칭 시 카카오 알림톡 발송 + 견적서 페이지 딥링크
  • 일·주 단위 가입자/의뢰 건수/매칭 성공률 통계 대시보드
  • 의뢰인 연락처 블라인드 처리 (개인정보 보호)
  • 소셜 로그인 (카카오 / 네이버 / Google)
  • AWS S3 라이프사이클 기반 대용량 파일 보관 아키텍처

제안 포인트

1. 도메인 정확도: "행정 워크플로우 + 자격증 인증" 직접 경험

본 프로젝트는 단순 매칭이 아니라 전문가 자격증 검증 → 의뢰 매칭 → 견적 비교 → 매칭 완료 → 리뷰의 5단계 워크플로우입니다. 유사 프로젝트(시니어 주간보호 관리 플랫폼)에서 보호자/요양보호사/관리자 3-actor + 자격 인증 + 알림 워크플로우를 풀스택으로 구축한 경험을 그대로 적용합니다.

2. 90일 내 단계별 납품 — 4·8·10·12주 마일스톤 준수

클라이언트 일정(4주 기획/디자인 컨펌, 8주 핵심 기능 개발 완료, 10주 통합 테스트, 12주 오픈)을 그대로 수용하여 단계별 산출물로 진행합니다. 각 마일스톤에서 화면 시연 + 산출물 문서를 함께 검수받아 후반 리스크를 미연에 차단합니다.

3. 외부 API 안정성 — 카카오 알림톡 + 딥링크 검증된 구현 패턴

카카오 알림톡 BizMessage API + 서명 토큰 기반 딥링크는 핀테크/SaaS 프로젝트에서 다회 구현한 검증된 패턴입니다. 발송 실패 시 SQS 재시도 큐 + LMS 폴백까지 자동화하여 "행정사가 즉시 인지" 요구사항을 충족합니다.

4. 장기 비용 최적화 — S3 라이프사이클 자동 전이

증거용 이미지/영상/대용량 서류의 장기 보관 비용을 60~70% 절감하는 S3 Standard → Standard-IA → Glacier Instant Retrieval 자동 전이 정책을 적용합니다. 데이터 손실 방지를 위한 Cross-Region Replication + 버전 관리도 기본 포함합니다.

5. 산출물 완성도 우선 — 클라이언트 #1 우선순위 직접 매핑

클라이언트가 명시한 우선순위(#1 산출물 완성도, #2 일정 준수, #3 금액)에 맞춰, 산출물 기준(스토리보드·디자인 원본·ERD·아키텍처 다이어그램·소스 코드·관리자 매뉴얼·유지보수 가이드) 완성도를 가장 우선으로 관리합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

행정 업무(음주운전 구제, 인허가, 비자, 외국인등록 등)는 일반 소비자가 어디서 누구에게 의뢰해야 하는지 정보가 파편화되어 있고, 견적도 행정사마다 천차만별인 시장입니다. 본 플랫폼은 의뢰인이 한 번의 의뢰서 작성으로 복수 행정사로부터 견적을 받고, 별점·리뷰·1:1 메시지를 통해 투명하게 비교·선택하는 견적 비교 플랫폼을 구축합니다.

최종 목표는 "행정 시장의 정보 비대칭성 해소 + 투명한 견적 제공"이며, 90일 내 안정적인 코어 MVP를 오픈하여 시장 검증을 진행할 수 있는 수준의 산출물을 납품합니다.

2. 사용자 정의

사용자 유형설명주요 기능
의뢰인 (Client) 행정 업무가 필요한 일반 소비자 회원가입, 의뢰서 작성, 견적 비교, 1:1 메시지, 별점 리뷰, 알림톡 수신
행정사 (Scrivener) 자격증 보유 전문 행정사 자격증 업로드, 프로필 관리, 신규 의뢰 알림 수신, 견적서 제출, 1:1 메시지
관리자 (Admin) 플랫폼 운영자 통계 대시보드, 회원·행정사 관리, 자격증 승인/반려, 매칭 모니터링, 알림톡 템플릿 관리

3. 기능 요구사항

3.1 의뢰인 (Client)

  • [필수] 통합 회원가입 (이메일 + 비밀번호) 및 소셜 로그인 (카카오 / 네이버 / Google)
  • [필수] 카테고리(음주운전·인허가·비자 등) 선택 기반 의뢰서 작성
  • [필수] 의뢰서 텍스트 입력 + 파일 업로드 (PDF/이미지/영상, S3 multipart)
  • [필수] 수신된 행정사 견적 비교함 (금액·프로필·리뷰 별점 요약)
  • [필수] 행정사 선정 전 1:1 메시지(채팅/문의) 송수신
  • [필수] 매칭/상담 완료 후 별점 + 텍스트 리뷰 작성
  • [필수] 카카오 알림톡 수신 (견적 도착 시)

3.2 행정사 (Scrivener)

  • [필수] 행정사 전용 회원가입 + 사업자등록증·자격증 파일 업로드
  • [필수] 전문 분야(다중 카테고리) + 상세 경력 + 포트폴리오 프로필 관리
  • [필수] 본인 전문 분야와 일치하는 신규 의뢰 발생 시 실시간 알림톡 수신
  • [필수] 의뢰 상세 내용 확인 (의뢰인 연락처 블라인드 처리)
  • [필수] 예상 청구 비용·업무 처리 기간·상세 제안 메시지 포함 견적서 제출
  • [필수] 의뢰인과의 1:1 메시지(채팅/문의) 송수신

3.3 관리자 (Admin)

  • [필수] 일간/주간 가입자, 의뢰 건수, 매칭 성공률 통계 대시보드
  • [필수] 일반 회원 및 행정사 계정 관리 (검색·정지·삭제)
  • [필수] 행정사 가입 시 제출한 자격증 뷰어 + 승인/반려(사유 입력)
  • [필수] 의뢰서 내용, 오고 간 견적 비용, 매칭 프로세스 데이터 모니터링
  • [필수] 알림톡 템플릿 발송 트리거 관리

3.4 공통/인프라

  • [필수] JWT 기반 인증 + RBAC 권한 분리 (의뢰인/행정사/관리자)
  • [필수] AWS S3 라이프사이클 자동 전이 (Standard → Standard-IA → Glacier Instant Retrieval)
  • [필수] 카카오 알림톡 BizMessage API + 서명 토큰 기반 딥링크
  • [필수] 알림톡 발송 큐(SQS) + 실패 재시도 + LMS/SMS 폴백
  • [필수] Cross-Region Replication 기반 백업
